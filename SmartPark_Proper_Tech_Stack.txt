SmartPark - Proper Technical Stack Document1. Vision Layer (Edge Processing Simulation)Language: Python 3.10+Computer Vision Libraries:- OpenCV (frame processing and visualization)- YOLOv8 (Ultralytics) for vehicle detection- ByteTrack for multi-object trackingModel Configuration:- yolov8n.pt for local laptop testing- yolov8s or higher for GPU-enabled deployment2. Backend Layer (Central System)Framework: FastAPI (async, high-performance API server)ORM: SQLAlchemyDatabase (Development Phase): SQLiteDatabase (Deployment Phase): PostgreSQLResponsibilities:- Receive entry/exit events from vision services- Maintain floor-wise vehicle counts- Calculate real-time slot availability- Provide floor recommendation API3. Real-Time CommunicationPrototype Phase: REST API pollingProduction Phase: WebSockets (FastAPI built-in support)4. Frontend (Admin Dashboard)Framework: ReactStyling: Tailwind CSSAPI Communication: AxiosFeatures:- Real-time floor occupancy table- Available slot display- Event log viewer- Floor recommendation status5. API LayerPOST /event - Receive entry or exit eventsGET /floors - Get floor-wise occupancy statusGET /recommend - Get optimal floor recommendationGET /events - Retrieve event logs6. Deployment LayerLocal Development: Run services via terminalContainerization: Docker and Docker ComposeProduction Deployment Options:- VPS (DigitalOcean)- Render- Railway7. Development Tools- Git and GitHub- Postman (API testing)- pgAdmin (PostgreSQL management)- VS Code (development environment)8. System Architecture OverviewCamera (RTSP or MP4)?
